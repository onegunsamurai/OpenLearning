# ADR-152: Isolated Docker Environments for Parallel Claude Code Worktrees

## Status
Proposed

## Context

When running `/pipeline` or E2E tests inside a git worktree (`.claude/worktrees/issue-<N>`),
the Docker dev environment fails due to four interrelated problems:

1. **Port conflicts** — all worktrees try to bind the same host ports (3000, 8000, 5432)
2. **CORS mismatches** — the backend's `CORS_ORIGINS` does not include non-standard frontend ports
3. **Symlinked `node_modules`** — git worktree sharing causes symlinks that break Turbopack
4. **Stale Docker volumes** — anonymous volumes from previous runs cache broken `node_modules`

These failures require 15-30 minutes of manual troubleshooting per worktree, breaking the
autonomous agent pipeline and blocking parallel development.

### Constraints

- Existing `docker-compose.yml` and `docker-compose.dev.yml` must not be modified.
- Backend already reads `CORS_ORIGINS`, `FRONTEND_URL`, `DATABASE_URL` from env vars (`backend/app/config.py`).
- Frontend's `NEXT_PUBLIC_API_URL` is configurable via env var.
- Playwright reads `BASE_URL` and `API_URL` from env vars (`frontend/playwright.config.ts`).
- `init_db()` in `backend/app/db.py` creates all tables on startup — no migration step needed.

### Decisions Already Made (from OQ review)

| ID | Decision |
|----|----------|
| OQ-1 | Validate and fail for port overflow (>65535). No modular arithmetic. |
| OQ-2 | Default to dev mode (volume mounts + hot reload). `MODE=prod` flag available. |
| OQ-3 | Symlink handling only in `worktree-dev.sh`, not in `worktree-create.sh`. |
| OQ-4 | Docker teardown integrated into `worktree-remove.sh`. |
| OQ-5 | `init_db()` handles fresh DB setup. No separate migration step. |
| OQ-6 | Override file lives inside the worktree dir. Absolute paths for all `-f` flags. |
| OQ-7 | Auto-configure Playwright env vars. Add `worktree-e2e` target. |

## Decisions

### D1: Port Derivation Formula

| Service  | Formula       | Example (N=144) | Internal port |
|----------|---------------|-----------------|---------------|
| Frontend | `3000 + N`    | `3144`          | `3000`        |
| Backend  | `8000 + N`    | `8144`          | `8000`        |
| Database | `5432 + N`    | `5576`          | `5432`        |

Validation: all derived ports must be within 1024-65535. Max issue number is 57535.
Script rejects N < 1 and any N where a derived port exceeds 65535.

### D2: Docker Compose Override Layering

**Dev mode (default):**
```
docker compose \
  -f <REPO_ROOT>/docker-compose.yml \
  -f <REPO_ROOT>/docker-compose.dev.yml \
  -f <WORKTREE_DIR>/docker-compose.worktree.yml \
  -p openlearning-wt-<N> \
  up -d --build --force-recreate -V
```

**Prod mode (`MODE=prod`):** Same but without `docker-compose.dev.yml`.

The worktree override is always last so it takes precedence on conflicting keys.
`--force-recreate -V` ensures stale anonymous volumes are cleaned.
`-p openlearning-wt-<N>` isolates containers, networks, and named volumes.

### D3: Generated Override File

**Location:** `<WORKTREE_DIR>/docker-compose.worktree.yml`

```yaml
# AUTO-GENERATED by scripts/worktree-dev.sh — do not edit
services:
  db:
    ports:
      - "127.0.0.1:<DB_PORT>:5432"
  backend:
    ports:
      - "127.0.0.1:<B_PORT>:8000"
    environment:
      CORS_ORIGINS: '["http://localhost:<F_PORT>"]'
      FRONTEND_URL: "http://localhost:<F_PORT>"
  frontend:
    ports:
      - "127.0.0.1:<F_PORT>:3000"
    environment:
      NEXT_PUBLIC_API_URL: "http://localhost:<B_PORT>"
```

In dev mode, volume mounts pointing to the worktree source directories are also added
to ensure hot-reload reflects worktree changes, not main repo changes.

Properties: no secrets, idempotent (regenerated each run), minimal (only port/URL overrides),
ports bound to 127.0.0.1 (per SR-04 from threat model).

### D4: File Organization

**New files:**
| File | Purpose |
|------|---------|
| `scripts/worktree-dev.sh` | Orchestration: validate, generate override, symlinks, Docker start, health check |
| `scripts/test-worktree-dev.sh` | Unit tests for port derivation and override generation (no Docker needed) |
| `docs/worktree-dev.md` | User-facing documentation |

**Modified files:**
| File | Change |
|------|--------|
| `scripts/worktree-remove.sh` | Add Docker teardown before `git worktree remove` |
| `scripts/worktree-list.sh` | Add Docker status column |
| `Makefile` | Add `worktree-dev`, `worktree-dev-down`, `worktree-e2e` targets |

**Not modified:** `docker-compose.yml`, `docker-compose.dev.yml`, `backend/app/config.py`,
`backend/app/main.py`, `frontend/playwright.config.ts`, `scripts/worktree-create.sh`, `.gitignore`.

### D5: worktree-dev.sh End-to-End Flow

```
 1. Parse arguments (ISSUE, MODE)
 2. Resolve REPO_ROOT (via git rev-parse)
 3. Validate: ISSUE >= 1, ports within range, Docker running, worktree dir exists, .env files present
 4. Handle symlinks: remove node_modules symlink, replace package-lock.json symlink with real copy
 5. Run npm install if node_modules missing
 6. Generate docker-compose.worktree.yml
 7. docker compose up -d --build --force-recreate -V
 8. Wait for health checks (--wait or manual polling, 120s timeout)
 9. Print URLs on success, dump logs on failure
```

Auto-detection (AC-11): when ISSUE is not provided, extract N from pwd pattern `.claude/worktrees/issue-<N>`.

### D6: Docker Teardown Integration

`worktree-dev-down`: `docker compose ... down` (preserves volumes by default, `VOLUMES=yes` to remove).
`worktree-remove.sh`: checks for running stack via project name, runs `docker compose -p openlearning-wt-<N> down -v --timeout 10`.

### D7: E2E Integration

`worktree-e2e`: ensures stack is running, exports `BASE_URL` and `API_URL`, runs `npx playwright test`.

### D8: worktree-list.sh Docker Status Column

Adds "Docker" column (`up`/`down`) using `docker compose -p openlearning-wt-<N> ps --quiet`.

## CLI API Contracts

### `make worktree-dev ISSUE=N [MODE=dev|prod]`
- Exit 0: all services healthy
- Exit 1: validation failure
- Exit 2: health check timeout (120s, logs dumped)
- Exit 3: Docker Compose error

### `make worktree-dev-down ISSUE=N [VOLUMES=yes]`
- Exit 0: stack stopped (or was already stopped)
- Exit 1: validation failure

### `make worktree-e2e ISSUE=N`
- Exit 0: all E2E tests passed
- Exit 1: stack startup failure
- Non-zero: Playwright exit code forwarded

## Consequences

### Positive
- Zero-config parallel dev environments with a single command
- No modifications to base Docker Compose files or backend code
- Predictable port scheme (issue 144 = port 3144)
- Enables fully autonomous agent pipeline for E2E testing
- Integrated lifecycle (worktree-remove cleans up Docker)

### Negative
- Port range consumption (max issue 57535)
- Docker resource usage (~1GB RAM per stack, ~10 simultaneous stacks practical limit)
- New ~200-line bash script to maintain

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| `--wait` unavailable on older Docker Compose | Low | Medium | Version detect, fall back to polling |
| Port collision with non-worktree process | Medium | Low | Catch error, suggest `lsof -i :<port>` |
| Dev mode volume mounts point to wrong dir | Medium | High | Override explicitly sets worktree paths |
| `.env` symlinks broken | Low | Medium | Validation warns with actionable message |
